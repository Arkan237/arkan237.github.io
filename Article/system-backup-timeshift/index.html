Membuat Backup Sistem dengan Timeshift di Linux

Ditulis pada 10 Agustus 2022 oleh Arkan

Membuat backup data tentunya adalah suatu hal yang sangat penting. Hal ini dilakukan untuk meminimalisir terjadinya kehilangan data. Seperti misalnya, komputernya terkena virus ataupun terjadi kerusakan pada harddisk, sehingga datanya tidak bisa diakses, dan lain sebagainya. Untuk membuat backup data biasa, bisa dilakukan dengan cara menyalin data ke tempat yang lebih aman (seperti misalnya di flash drive). Namun ada juga hal yang sebaiknya kalian lakukan juga, yaitu membuat backup sistem.

Mengapa kita perlu membuat backup sistem?

Sebenarnya membuat cadangan untuk data pribadi kita sendiri juga sudah cukup. Namun yang artinya, jika kalian melakukan install ulang OS kalian, maka kalian perlu melakukan konfigurasi ulang pada komputer kalian yang tentunya akan memakan waktu yang lama. Dan melakukan konfigurasi ulang sistem pada Linux adalah hal yang sangat merepotkan dan menjadi mimpi buruk bagi kebanyakan pengguna Linux. Oleh karena itu, kita perlu program yang dapat melakukan backup sistem dengan mudah dan dapat di restore dengan mudah juga.

Apakah ada program semacam itu? Tentu saja ada. Nama programnya sudah penulis tulis di judul artikel ini, yaitu Timeshift.

Timeshift, program untuk membuat backup sistem dengan mudah di Linux

Timeshift adalah program yang dapat kalian gunakan untuk membuat backup sistem di Linux. Timeshift dibuat dan dikembangkan oleh Tony George. Program ini bersifat open source dan tentunya gratis. Yang artinya, kalian tidak perlu mengeluarkan uang untuk dapat menggunakan program ini. Program ini mirip dengan program System Restore pada OS Jendela dan Time Machine pada OS Buah.

Berbeda halnya dengan OS Jendela dan Buah, di Linux tidak terdapat program untuk membuat cadangan sistem dengan mudah secara bawaan. Sehingga, aplikasi Timeshift ini bersifat third-party software (perangkat lunak pihak ketiga). Namun ada juga distro yang sudah menyediakan aplikasi Timeshift secara bawaan.

Karena program ini bersifat open source, maka kalian dapat melihat source code dari program timeshift. Kalian juga dapat ikut berkontribusi pada pengembangan timeshift. Untuk melakukannya, kalian dapat langsung melihat repository nya pada GitHub.

Instalasi Timeshift

Karena tidak tersedia secara bawaan, maka kita perlu meng-install nya secara manual. Kita dapat meng-install nya menggunakan package manager bawaan dari distro yang digunakan.

Untuk distro Ubuntu (20.04+) :

sudo apt install timeshift

Untuk distro Fedora :

sudo dnf install timeshift

Untuk distro Arch, kalian dapat meng-install nya melalui AUR. Kalian dapat meng-install nya secara manual atau menggunakan AUR Helper favorit kalian. Sebagai contoh, penulis menggunakan yay :

yay -S timeshift

Mengonfigurasi Timeshift

Setelah kalian selesai meng-install, maka kalian dapat mencoba untuk membukanya melalui menu aplikasi yang tersedia pada DE yang kalian gunakan. Setelah kalian membukanya, sistem akan menanyakan password user kalian. Hal ini terjadi karena timeshift membutuhkan hak akses root untuk dapat berjalan dengan baik.

Setelah kalian memasukan password user kalian, maka timeshift akan langsung terbuka. Dan kalian akan disambut dengan tampilan seperti pada gambar. Bagian ini disebut dengan setup wizard. Disini kalian dapat mengonfigurasi timeshift dengan mudah. Kalian hanya perlu membaca keterangan yang ada dan menekan tombol Next.

Tips
Jika dalam komputer/laptop kalian terdapat lebih dari 1 storage drive. Penulis menyarankan kalian untuk menyimpan snapshot timeshift pada storage drive satunya. Hal ini dilakukan agar ketika storage drive utama kalian rusak, maka snapshot nya tetap tersimpan dengan aman dan kalian dapat memulihkannya ketika kalian sudah mengganti storage drive utamanya.

Membuat snapshot pada timeshift

Snapshot adalah kumpulan data sistem yang sudah dicadangkan oleh timeshift. Untuk membuatnya, kalian hanya perlu mengklik tombol Create. Maka timeshift akan membuatkan snapshot secara otomatis.

Kapan saya harus membuat snapshot di timeshift?

Sebenarnya tidak ada aturan mengenai kapan kalian harus membuat snapshot di timeshift. Kalian bebas menentukan kapan kalian harus membuat snapshot. Mau itu setiap bulan, minggu, hari, atau bahkan setiap kali kalian menyalakan komputer, tidak masalah.

Kalau penulis sendiri, lebih suka membuat snapshot secara manual sebelum penulis melakukan update sistem. Apalagi penulis sendiri menggunakan distro rolling release. Hal tersebut dilakukan agar kalau misalnya terdapat masalah setelah update, penulis tidak perlu repot-repot mencari tahu cara memperbaiki masalahnya. Cukup buka timeshift lalu tinggal restore deh.

Selain membuat snapshot secara manual, penulis juga mengatur timeshift agar membuat snapshot secara otomatis setiap bulannya. Hal tersebut dilakukan untuk berjaga-jaga kalau misalnya penulis melakukan suatu hal dan lupa membuat snapshot. Agak ribet soalnya kalau harus menyelesaikan masalahnya sendiri.

Restore snapshot dari Timeshift

Misalnya suatu ketika kalian mengalami masalah pada komputer kalian. Untungnya, kalian sudah membuat snapshot di Timeshift. Namun bagaimana cara me-restore nya? Caranya cukup mudah. Kalian hanya perlu membuka Timeshift, lalu memilih snapshot yang mau di-restore, setelah itu klik tombol Restore.

Namun bagaimana kalau rusaknya agak parah sampai komputernya tidak bisa masuk ke OS nya? Tenang saja, di Linux ada yang namanya Live boot. Kalian bisa me-restore snapshot Timeshift dari sana. Bagaimana caranya? Pertama kalian perlu masuk ke Live boot Linux yang kalian gunakan (atau menggunakan distro yang berbeda juga tidak masalah seharusnya). Setelah itu, install Timeshift dalam mode Live boot. Setelah itu, lakukan cara yang sama seperti me-restore snapshot ketika komputer kalian bisa booting dengan normal.

Jika kalian menyimpan snapshot nya di dalam external drive (seperti di HDD External, Flash Drive, dsb), sebaiknya kalian colokan dulu external drive nya ke komputer kalian sebelum kalian membuka Timeshift. Hal tersebut dilakukan agar Timeshift dapat langsung menemukan snapshot yang tersimpan.

Kesimpulan

Menurut penulis sendiri, Timeshift adalah life saver. Karena dengan menggunakan Timeshift, kita tidak merasa stress karena harus melakukan konfigurasi ulang pada sistem kita. Tinggal restore snapshot yang sudah kita buat, dan semuanya akan kembali seperti sebelum masalah terjadi tanpa harus dikonfigurasi ulang. Timeshift ini juga sangat berguna jika kalian sudah stuck pada suatu masalah dan jalan terakhirnya adalah install ulang. Penulis sendiri juga sering menggunakan Timeshift untuk menyelesaikan masalah yang ada pada komputer penulis.
