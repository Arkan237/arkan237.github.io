Cara Menghapus Snap Di Ubuntu 22.04

Ditulis pada 7 Agustus 2022 oleh Arkan

Di dalam Ubuntu, terdapat 2 package manager, yaitu apt dan snap. Apa perbedaannya? Sederhananya, apt adalah package manager tradisional yang bersifat native. Artinya aplikasi yang kalian install melalui package manager ini, dijalankan langsung pada sistem kalian (tidak dijalankan pada container yang dapat menambah beban pada sistem kalian). Namun ada beberapa masalah pada package manager ini. Oleh karena itu, dibuatlah yang namanya Universal Package Manager yang dapat mengatasi masalah yang disebabkan oleh package manager tradisional. Snap adalah salah satu contoh dari Universal Package Manager.

Snap dibuat oleh pengembang Ubuntu sendiri, yaitu Canonical. Snap ditujukan sebagai alternatif yang lebih baik dari apt. Namun banyak pengguna Linux yang membenci snap. Mengapa demikian? Bukannya lebih bagus snap daripada apt? Hal tersebut terjadi karena snap mempunyai masalahnya tersendiri dan masalahnya ini, membuat banyak pengguna Linux yang terganggu. Masalahnya adalah snap itu lambat. Ketika kalian mencoba memasang aplikasi melalui snap, tentunya waktu yang dibutuhkan jauh lebih lama dibandingkan dengan kalian memasangnya melalui apt. Dan juga, ketika kalian menjalankan aplikasi yang di install melalui snap, tentunya butuh waktu lama untuk menjalankan aplikasi itu. Bukan hanya itu, ukuran aplikasi yang di install melalui snap jauh lebih besar. Sehingga jika kalian banyak memasang aplikasi melalui snap, penyimpanan komputer kalian akan cepat penuh. Oleh karena itu, alangkah baiknya kalian menghapus snap pada sistem kalian. Berikut ini adalah langkah-langkah menghapus snap pada Ubuntu 22.04.

Catatan
Tutorial ini memang dibuat untuk Ubuntu versi 22.04 LTS. Namun tutorial ini seharusnya juga dapat diterapkan pada Ubuntu versi baru atau lama.

Langkah 1 : Cek package snap yang terpasang

Sebelum menghapus snap, kalian perlu mengecek apakah di dalam sistem kalian terdapat package snap yang terpasang. Cara mengeceknya, jalankan command di bawah ini pada terminal.

snap list

Langkah 2 : Hapus package snap yang terpasang

Jika terdapat package snap yang terpasang di dalam sistem kalian, maka kalian perlu menghapusnya satu-persatu terlebih dahulu. Cara menghapusnya adalah dengan menjalankan command sebagai berikut

sudo snap remove --purge package-name

Kata package-name dalam command di atas perlu diganti dengan nama package yang terpasang. Misalnya, kalian ingin menghapus firefox. Maka command yang perlu kalian masukan adalah

sudo snap remove --purge firefox

Langkah 3 : Uninstall snap

Setelah kalian menghapus package snap yang terpasang. Maka langkah selanjutnya adalah menghapus snap itu sendiri. Untuk melakukannya, jalankan command dibawah ini

sudo apt purge snapd

Langkah 4 : Hapus config snap

Setelah kalian mengahpus snap, snap akan meninggalkan file config nya di dalam direktori /home/$USER/snap. Untuk menghapusnya, jalankan command dibawah ini

rm -rf ~/snap

Langkah 5 : Cegah agar snap tidak terpasang kembali

Setelah kalian membersihkan sistem kalian dari snap, ada satu langkah terakhir yang perlu kalian lakukan. Yaitu mencegah agar snap tidak terpasang kembali. Hal ini dilakukan agar snap tidak terpasang secara otomatis pada sistem kalian tanpa kalian sadari. Bisa saja mungkin suatu hari ketika kalian melakukan update sistem, snap akan otomatis terpasang kembali tanpa kalian sadari. Untuk mencegah hal tersebut terjadi, maka kita perlu mencegah agar snap tidak terpasang kembali. Untuk melakukannya, jalankan command berikut

sudo apt-mark hold snapd

Apakah akan ada masalah yang muncul setelah saya menghapus snap?

Mungkin kalian akan bertanya seperti itu ketika kalian membaca artikel ini. Sejauh ini, penulis belum menemukan masalah yang serius setelah penulis mencoba menghapus snap pada Ubuntu. Namun penulis menemukan satu masalah yang terjadi. Yaitu kalian tidak bisa memasang package yang memerlukan snap agar bisa terpasang di Ubuntu. Contohnya adalah Firefox dan Chromium.

Hal tersebut terjadi karena package Firefox dan Chromium itu sebenarnya hanya package kosong di dalam repository Ubuntu. Di dalamnya hanya terdapat script untuk memasang Firefox atau Chromium melalui snap. Karena package snapd diatur agar tidak bisa terpasang, maka kita tidak bisa memasang package Firefox dan Chromium di Ubuntu melalui apt. Karena kedua package tersebut membutuhkan snap agar bisa terpasang.

Cara mengatasi masalah tersebut tanpa perlu memasang package snapd adalah dengan memasang Firefox atau Chromium secara manual menggunakan file deb. Cara tersebut kurang direkomendasikan karena akan menyulitkan kalian ketika kalian ingin melakukan update pada kedua package tersebut. Cara lainnya adalah dengan menggunakan Flatpak. Ubtuk tutorial pemasangannya, silahkan kalian lihat di situs resminya atau mungkin nanti penulis akan buat tutorialnya.

Apakah saya perlu menghapus snap pada sistem saya?

Penulis membuat tutorial ini untuk kalian yang merasa terganggu dengan snap. Kalian tidak wajib untuk mengikuti tutorial ini. Jika kalian memang membutuhkannya dan kalian tidak merasa terganggu, kalian tidak perlu menghapusnya. Namun penulis sendiri lebih suka memasang aplikasi melalui apt atau flatpak. Dan semenjak penulis menggunakan distro berbasis Arch, penulis lebih suka memasang aplikasi melalui AUR. Jadi jika kalian lebih suka memasang aplikasi melalui snap, kalian tidak perlu menghapusnya. Namun jika kalian jarang sekali menggunakan snap atau bahkan kalian membencinya, silahkan kalian ikuti tutorial ini untuk menghapusnya.
